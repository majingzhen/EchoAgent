## 二、第一梯队：核心可信度提升（建议优先）

### 2.1 沙盘推演 LLM 化

**优先级**：P1 — 系统可信度的最大短板

**现状问题**：
- 画像的行为（点赞/评论/转发/忽略）由随机数决定，不依赖画像人设
- 评论内容是模板拼接（`{name} 在第{round}轮评论：关注价格与效果`），无差异
- 模拟报告的洞察是硬编码文案，不随内容变化

**目标**：
- 每个画像的行为由 LLM 基于人设 + 内容 + 社交压力「决策」
- 评论内容由 LLM 根据画像性格生成自然语言
- 报告由 LLM 基于实际模拟数据分析生成

**改动范围**：
- `simulation_engine.py`：`_build_round_actions()` 从随机逻辑改为并发 LLM 调用
- `simulation_engine.py`：`_build_report()` 从硬编码改为 LLM 生成
- 需要 `asyncio.Semaphore` 控制并发（画像数 × 轮次 = 大量 LLM 调用）
- 前端无需改动（数据结构不变）

**关键设计**：
```
每轮每个画像：
  输入 = 画像人设 + 待测内容 + 前几轮的社交反馈（他人评论摘要）
  输出 = {"action": "评论", "comment": "...", "sentiment": 7.2, "purchase_intent": 0.6}
```

**风险**：
- LLM 调用量显著增加（6画像 × 8轮 = 48次），需控制 token 和并发
- 建议 light 模型(qwen-turbo) + 简短 prompt + Semaphore(5)

**预估工作量**：后端 1-2 天

---

### 2.2 历史会话管理 + 结果对比

**优先级**：P1 — 没有历史记录，每次都是一次性的

**现状问题**：
- 各模块没有历史列表页，用完找不到之前的结果
- 无法对比两次焦点小组、两次沙盘推演的结果差异

**目标**：
- 每个模块增加「历史记录」入口，按时间倒序展示
- 支持点击查看历史会话详情（只读回放）
- 支持选择两条记录做「对比视图」（侧对侧展示）

**改动范围**：
- 后端：各模块的 list 接口已有（如 `/api/sentiment-guard/sessions`），焦点小组和内容工坊需补充
- 前端：每个模块新增 `xxxHistory.vue` 页面 + 路由
- 前端：新增 `xxxCompare.vue` 对比页面（可选，焦点小组和沙盘优先）

**预估工作量**：前后端各 2 天

---

### 2.3 画像记忆系统

**优先级**：P2 — 提升虚拟消费者的连续性和可信度

**现状问题**：
- 画像每次回答都是无状态的，不记得之前说过什么
- 同一画像在焦点小组说「太贵了」，沙盘推演里却可能点赞「限时特价」

**目标**：
- 画像参与焦点小组/沙盘后，关键观点记录到「画像记忆」
- 后续互动时，prompt 注入历史记忆摘要，保持态度连续性
- 提供「画像态度轨迹」查看功能

**改动范围**：
- 新增 `persona_memory` 表（persona_id, session_type, session_id, memory_summary）
- `focus_group_service` / `simulation_engine` 完成后写入记忆
- `_build_persona_reply()` 等方法查询并注入历史记忆
- 前端画像详情页增加「历史记忆」tab

**数据结构**：
```sql
CREATE TABLE persona_memory (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    persona_id BIGINT NOT NULL,
    session_type VARCHAR(50) NOT NULL,  -- focus_group / simulation / workshop
    session_id BIGINT NOT NULL,
    memory_summary TEXT NOT NULL,       -- LLM 提取的关键观点摘要
    created_at DATETIME NOT NULL,
    INDEX idx_pm_persona_id (persona_id)
);
```

**预估工作量**：后端 1.5 天，前端 0.5 天

## 三、第二梯队：功能延展

### 3.1 视觉素材测试（焦点小组增强）

**目标**：焦点小组支持上传图片（产品图/海报/详情页截图），画像对视觉素材发表看法

**方案**：
- 前端：提问时支持附带图片 URL 或 base64
- 后端：使用多模态 LLM（qwen-vl-plus）解析图片 + 画像人设生成反馈
- 需要 LLM 客户端增加 vision 能力

**依赖**：通义千问 VL 模型 API

**预估工作量**：后端 1.5 天（LLM client 扩展 + service 改造），前端 0.5 天

---

### 3.2 跨模块工作流编排引擎

**目标**：把 7 个模块串成可配置的自动化流水线

**预定义工作流**：
- 「内容验证流」：内容工坊 → 沙盘推演 → (风险>7?) 舆情哨兵
- 「完整营销活动流」：市场智脑 → 画像工厂 → 内容工坊 → A/B沙盘 → 舆情风控 → 策略评审
- 支持用户自定义步骤和条件触发

**改动范围**：
- 新增 `WorkflowEngine` service，解析步骤 DAG 并依次执行
- 新增 `TriggerManager`，监听模块事件并触发关联流程
- 新增 `workflow_session` 表记录执行历史
- 前端：工作流可视化配置页 + 执行监控面板

**预估工作量**：后端 3 天，前端 2 天

---

### 3.3 报告导出（PDF / PPT）

**目标**：焦点小组总结、沙盘报告、策略参谋报告导出为可分享的文档

**方案**：
- 后端：使用 `weasyprint`（PDF）或 `python-pptx`（PPT）生成
- 每个模块的报告页增加「导出」按钮
- 导出内容包含图表（沙盘曲线、图谱截图等）

**预估工作量**：后端 2 天（模板 + 渲染），前端 0.5 天（按钮 + 下载）

---

### 3.4 画像组合推荐

**目标**：基于人群描述，AI 推荐最优画像组合方案

**方案**：
- 生成画像前，先用 LLM 分析目标人群，输出推荐的画像类型分布
- 例如：「8人组建议：价格敏感×3、品质导向×2、社交驱动×2、务实×1」
- 展示推荐理由，用户可调整后再生成

**预估工作量**：后端 0.5 天，前端 0.5 天

### 3.5 联网数据增强（AI Search）

**目标**：接入 Anspire AI Search，为市场智脑、舆情哨兵、策略参谋注入实时外部数据

**背景**：
- 当前各模块分析仅基于用户手动输入的文本，缺乏实时市场信息
- Anspire AI Search 提供实时网页搜索能力，百亿级网页数据源，标准版 ¥30/1000次

**API 信息**：
- Endpoint：`GET https://plugin.anspire.cn/api/ntsearch/search`
- 鉴权：`Authorization: Bearer {API_KEY}`
- 参数：query（查询词）、top_k（返回条数 10~50）、Insite（限定站点）、FromTime/ToTime（时间范围）
- 返回：网页结果列表，含标题、摘要、URL
- 参考文档：`docs/ai_search/`

**接入模块与场景**：

| 模块 | 场景 | 搜索策略 |
|------|------|---------|
| 市场智脑 | `build_graph` 时可选联网补充数据 | 用品牌名+行业关键词搜索，合并到 source_text 后再提取图谱 |
| 舆情哨兵 | 事件分析前搜索真实舆情 | 用事件描述关键词搜索近期新闻，注入风险评估 prompt |
| 策略参谋 | 生成建议前搜索行业背景 | 用问题关键词搜索，作为 context_info 补充 |

**不接入的模块**：画像工厂（虚构人物不需要）、焦点小组（讨论闭环）、沙盘推演（模拟闭环）、内容工坊（LLM 自身能力足够）

**改动范围**：
- 新增 `app/search/client.py`：封装 SearchClient，接口风格与 LLMClient 一致
- `config.py` 新增 `SearchConfig(api_key, base_url, enabled)`
- 市场智脑：`MarketService.build_graph()` 增加可选参数 `enable_search=True`
- 舆情哨兵：`SentimentGuardService` 分析前调用搜索补充真实舆情
- 策略参谋：`StrategyAdvisorService` 分析前搜索行业背景

**设计原则**：
- 联网搜索为可选增强，`enabled=False` 时所有模块行为不变
- 搜索结果作为 LLM prompt 的补充 context，不改变现有数据结构
  - 搜索失败不阻断主流程，降级为无联网模式

**预估工作量**：SearchClient 封装 0.5 天 + 3 个模块各 0.5 天 = 后端 2 天

## 四、第三梯队：生态方向（长期）

### 4.1 多语言 / 多地区市场

- 画像支持生成其他国家/地区的消费者（东南亚、欧美、日韩）
- 画像用目标语言回答
- 内容工坊支持多语言文案生成
- prompt 模板国际化

### 4.2 投放效果回收

- 预留真实广告平台数据对接口（巨量引擎 / 腾讯广告 / Meta Ads）
- 沙盘预测 vs 实际投放效果对比看板
- 反哺模型准确性评估

### 4.3 数字孪生消费者

- 画像升级为持久化 Agent，具备长期记忆和偏好演变
- 画像态度随多次互动逐步变化（首次了解 → 试用 → 忠诚/流失）
- 可构建长期跟踪的「虚拟消费者面板」

### 4.4 营销日历 + 自动排期

- 基于节日/电商节点/行业热点自动触发内容生产流程
- 618/双11/春节等关键节点提前N天自动启动「完整营销活动流」工作流
- 与工作流编排引擎联动

### 4.5 团队协作

- 多用户在同一项目上协作
- 角色分工：品牌经理（看报告）、内容运营（用工坊）、数据分析（看沙盘）
- 审批流：文案需品牌经理审核后才能进入沙盘测试
